アゴラ上の日銀管理ノードの「5大業務」1. トークン化日銀当座預金の発行（ミント）と回収（バーン）役割：民間銀行（MUFGなど）が保有する従来の日銀当座預金（ベースマネー）の残高に応じて、アゴラ共通レジャー上で動く「トークン化日銀当座預金」を1対1の価値でデジタル発行します。制御：民間銀行がデジタル円を通常の預金に戻したい場合は、ノード上でトークンを即座に消滅（バーン）させ、伝統的な日銀当座預金口座の残高へ払い戻します。2. スマートコントラクト（プログラムルール）の独占的統治役割：デジタル円が持つすべてのプログラムやルール（スマートコントラクト）の書き換え権限を100%独占管理します。制御：日銀以外の機関（民間銀行やBIS、他国の中央銀行）が、デジタル円のプログラムを勝手に変更したり、偽造したりできないよう、ノードの暗号鍵（マスターキー）を厳重に管理します。3. 「決済の最終性（ファイナリティ）」の即時承認役割：為替ノードを介して「MUFGのデジタル円」が「JPM（または為替ノード）」へ移転する際、そのトランザクション（取引データ）が正当なものであるかをミリ秒単位で検証・承認します。効果：日銀の管理ノードが署名（バリデーション）した瞬間に、法的に「取り消し不可能な確定決済（最終性）」が成立します。4. 流動性監視とマクロプルーデンス（市場安定化）役割：24時間365日動くシステムの中で、日本の民間銀行（メガバンク等）が「夜間や土日にデジタル円の流動性不足（残高不足）」に陥っていないかをリアルタイムに監視します。制御：急激な市場の混乱（取り付け騒ぎや他国通貨の暴落）が起きた際、日本の金融システムを守るために、一時的に取引を制限したり、緊急の流動性（デジタル円）を供給したりするコントロールセンターとして機能します。5. 参加ノードの認証とアクセス権限管理（ゲートキーパー）役割：アゴラ計画は誰でも参加できるパブリックチェーン（Bitcoin等）ではなく、許可された機関のみが参加する「認可型（パーミッションド）チェーン」です。制御：日銀は、どの民間銀行や為替ノードに「デジタル円を扱う権限」を与えるかを厳格に審査し、そのアクセス許可証（暗号証明書）の発行・管理を行います。不審な動きをしたノードの権限を即座に剥奪するのも日銀ノードの役割です。

中央銀行として「やらないこと」日銀が中立性と通貨の信頼性を保つため、このノードを通じて絶対に介入しない領域もあります。一般企業の取引データの閲覧：実需レイヤー（企業AやB）の具体的な取引内容や顧客の個人情報は、プライバシー保護技術（ゼロ知識証明など）によって隠蔽され、日銀ノードであっても原則として直接中身を見ることはできません（マネロン対策の暗号化された監査データのみを扱います）。為替レートの決定：前述の通り、為替リスクを負う取引（マーケットメイク）には一切関与せず、民間が勝手に動かした資金の「帳簿上の移動」を承認することだけに徹します。
